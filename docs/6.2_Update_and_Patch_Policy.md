# 6.2 Update & Patch Policy

---

## 1. Update Mechanism
*   **Current:** Manual Update. Users must download the latest `.exe` installer from the distribution source (GitHub Releases) and run it to overwrite the existing installation.
*   **Future:** Application will check for updates on startup via GitHub API and prompt the user to download.

## 2. Versioning Scheme
*   Adheres to **Semantic Versioning (SemVer 2.0.0)**: `MAJOR.MINOR.PATCH`.
    *   **Major:** Breaking UI/Architecture changes.
    *   **Minor:** New functionalities / Skills.
    *   **Patch:** Bug fixes / performance tweaks.

## 3. Security Patches
*   Critical security vulnerabilities (e.g., Remote Code Execution exploits) will be addressed with hotfix releases (e.g., 1.1.1) within 48 hours of discovery.

## 4. Version Support Policy
*   **Active Support:** Latest Major version (1.x).
*   **Security Only:** Previous Major version (0.x) for 6 months after deprecation.
*   **End of Life:** Versions older than 1 year are unsupported.
