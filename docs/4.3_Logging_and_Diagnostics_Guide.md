# 4.3 Logging & Diagnostics Guide

---

## 1. Log Locations
*   **Main Process (Startup/Launch):**
    *   Path: `%APPDATA%\assistant-rex\main_process_log.txt`
    *   Contains: Startup timestamp, Resources path, Backend spawn errors.
*   **Backend Output (Runtime):**
    *   Path: Console Output (Standard Out/Err) captured by Electron.
    *   Visible in: Terminal (if running from source) or DevTools Console (Ctrl+Shift+I in App).

## 2. Log Levels
*   **[INFO]:** Normal operation events (State changes, Tool calls).
*   **[WARNING]:** Non-critical failures (Audio buffer underrun, API rate limit).
*   **[ERROR]:** Critical failures (Process crash, Missing API Key).

## 3. Common Error Codes
*   **ERR_NO_API_KEY:** Backend failed to initialize Gemini Client. Set `GEMINI_API_KEY`.
*   **ERR_AUDIO_DEVICE:** PyAudio could not open input stream. Check Microphone privacy settings.
*   **ERR_CONNECTION_REFUSED:** Socket.IO handshake failed. Backend process likely died.

## 4. Log Format
Timestamp | Level | Component | Message
--- | --- | --- | ---
`2026-01-19T10:00:00Z` | `[INFO]` | `Main` | `Spawning backend process...`

## 5. How Users Export Logs
1.  Navigate to `%APPDATA%\assistant-rex` in File Explorer.
2.  Copy `main_process_log.txt`.
3.  Attach this file to any Incident Report or Support Ticket.
